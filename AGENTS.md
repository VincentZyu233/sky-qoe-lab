# 仓库智能体说明

## 适用范围

- 本说明适用于整个仓库。
- 本仓库用于开发仅供私人使用的 Sky 体验质量模组，以及配套加载器、CE 桥接器、诊断工具和研究文档。
- 优先实现本地体验改善与调试功能，不得加入请求洪泛、拒绝服务行为或不必要的服务器流量。

## 提交共同作者

- 本仓库今后的每一个提交都必须将 ra1nyxin 和 Codex 列为共同作者。
- 每条提交信息末尾必须先空一行，再原样加入以下两条落款：

  `Co-authored-by: ra1nyxin <newrainyxin@outlook.com>`

  `Co-authored-by: Codex <codex@openai.com>`

- 智能体创建的文档提交、修复提交、整理提交、发布提交和合并提交都不得省略、修改或替换该落款。
- 用户要求的其他作者或共同作者落款也必须保留。

## 工作方式

- 开始逆向分析或运行时操作前先阅读根目录 `README.md`；其中记录已确认的相对虚拟地址、偏移、特征、工具、失败方向和运行时观察。
- 将有长期价值的新发现和失败方向写入根目录 `README.md`，避免后续会话重复踩坑。
- 修改范围应紧贴当前任务，并保留脏工作区中与任务无关的用户改动。
- 文本搜索优先使用 `rg`，手动修改文本文件优先使用 `apply_patch`。
- 源码和文档应纳入 Git；除非用户明确要求，否则构建目录、缓存、转储和临时截图不得入库。
- 不得提交 `Sky_dump.bin`、`Sky_full_dump.bin`、进程转储、CE 数据库、凭据或本机缓存目录。

## 项目目录

- `SkyQoEMenu/` 包含注入式 Win64 ImGui 菜单、运行时状态接口、游戏钩子、测试和单文件加载器源码。
- `CeBridge/` 包含本机 Cheat Engine 自动化桥接器及请求脚本。
- `SkyOutfitReader/` 包含外部穿搭读取实验及辅助工具。
- `docs/` 包含计划、研究资料和适合纳入仓库的预览图片。

## 运行安全

- 所有自动功能必须默认关闭，尤其是循环传送烛火和循环生成本地特效。
- 将游戏交还用户前不得留下已启用的断点。
- 执行运行时写入前，必须重新获取并校验当前 Avatar、Outfit、Transform、游戏构建和相关指针归属。
- 调用游戏自身的修改函数时，应复用已经建立的游戏线程钩子，不得直接从 Overlay、HTTP、CE 或任意远程线程调用。
- 测试服饰或位置修改时，应先记录原始状态并在结束后恢复，除非用户明确要求保留修改。
- Sky 无响应时，应先停止自动功能并尽可能安全卸载菜单；不要持续进行未经验证的盲写，必要时干净重启游戏。

## 验证要求

- C++ 改动必须在源码树外构建，不得提交生成的二进制文件。
- 修改菜单或加载器后，必须运行动态库接口测试和 `SkyQoELoader.exe --check --quiet`。
- Sky 正在运行时，应确认 `/health` 返回预期版本，并满足 `buildSupported=true` 与 `stateValid=true`。
- 注入或重载后，应确认 Sky 保持响应且所有自动功能均为关闭状态。
- 每次提交前必须运行 `git diff --check` 并检查 `git status`。
